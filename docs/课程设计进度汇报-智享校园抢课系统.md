# 基于 Java 的 Web 高并发应用程序——课程设计进度汇报（修订稿）

## 一、所选题目

本次课程设计选题为**基于 Java 的 Web 高并发应用**，项目名称为**智享校园 —— 热门公选课抢课系统**，面向校园场景中「**大量学生同时争抢有限热门选修课名额**」的典型高并发问题，重点验证**读性能**与**写一致性**（不超卖、防重复选课）。

## 二、开发工具

| 类别 | 具体工具 / 技术 | 用途说明 |
|------|-----------------|----------|
| 后端 | IntelliJ IDEA、JDK 8（pom 配置）、Spring Boot 2.6 | REST 接口、事务与业务编排；JPA（Hibernate）访问 PostgreSQL |
| 前端 | Node.js、Vue 2 + Element UI | 学生端抢课与管理端教务操作界面 |
| 数据库 | PostgreSQL（pgAdmin 4 等工具管理） | 持久化课程、学生、选课、抢课日志、系统配置等 |
| 缓存 | Redis | 课程单课缓存、列表缓存等，减轻高并发读压力 |
| 测试 / 调试 | JMeter、Postman | 并发压测、接口联调 |

*说明：若课程要求写 JDK 17，以本机实际编译运行版本为准，并在报告中与 pom 中 java.version 保持一致说明。*

## 三、设计思路

### 1. 核心目标

- **读得快**：课程列表等高频查询尽量走 **Redis**，降低 PostgreSQL 压力。
- **抢得准**：抢课扣库存采用 **数据库层原子更新 + 乐观锁式条件更新**（`WHERE left_stock > 0` 再减 1），避免超卖。
- **防重抢**：选课关系表上对 **学生 + 课程** 做唯一约束或存在性校验，避免重复插入选课记录。
- **可运营**：管理端维护课程、抢课时间窗/开关、退课截止时间；支持选课名单 **Excel 导出**。

### 2. 功能模块（与实现对应）

**（1）用户端（学生）**

- **登录**：**学号 + 密码**（如 MD5 校验），登录成功后前端使用返回的学生标识（如 studentId / 学号）参与后续抢课、我的课程等请求（**未采用 JWT**，与常见 Session / 请求参数携带身份类课程项目一致）。
- **课程列表**：支持筛选与展示；列表数据可结合 **Redis 缓存** 降低数据库读放大。
- **抢课**：同步请求后端；在**抢课开放时间段且开关开启**时，执行**库存扣减 + 插入选课记录 + 抢课日志**；失败场景返回明确原因（未开放、已选过、名额已满等）。
- **我的课表 / 退课**：查询 student_course 等表展示已选课程；退课时**恢复库存**，并受**全局退课截止时间 / 开关**等业务规则约束；同步更新 **Redis** 中的课程缓存。

**（2）管理端（教务）**

- **管理员登录**：独立账号体系（如 admin 表）。
- **课程管理**：课程 CRUD、分页与多条件查询（课程名、分类、剩余名额等）；**新增/修改后写回 Redis** 做缓存预热或更新。
- **抢课配置**：通过 **system_config**（或等价配置表）维护抢课**开关**、**起止时间**、**退课截止时间**等，与学生端、退课逻辑共用。
- **选课统计与导出**：按课程统计选课人数等；使用 **EasyExcel** 导出 Excel 名单。

### 3. 高并发相关技术要点

| 方案 | 解决问题 | 典型范围 |
|------|----------|----------|
| Redis 缓存 | 热点课程信息、列表查询读放大 | 读多写少场景 |
| 条件更新扣库存（乐观思路） | 并发下库存扣减一致、防止超卖 | 抢课写路径 |
| 唯一约束 / 存在性校验 | 同一学生重复选同一门课 | 选课记录写入 |
| 抢课日志 | 可追溯每次抢课结果，便于分析与排错 | 写路径旁路记录 |

## 四、进展情况（结题版）

### 1. 已完成内容（汇总）

- **工程与运行环境**：Spring Boot 工程骨架；集成 **PostgreSQL、Redis、JPA、RedisTemplate**；Web 服务（如 8080）；前端 8081 等与后端联调。
- **数据模型**：**course**（课程与库存）、**student**（学号登录）、**student_course**（选课关系）、**admin**、**system_config**、**seckill_log**（抢课日志）等，支撑业务闭环。
- **核心业务**：学生登录、课程列表与抢课、我的课程与退课（含配置校验）；管理端课程管理与抢课配置；选课统计与 **Excel 导出**。
- **高并发相关实现**：Redis 课程缓存与更新；抢课 reduceStock 类原子扣减 + 事务内插入选课；重复选课校验；全局异常处理统一错误返回，便于前后端联调。
- **问题排查与优化**：分页与 PostgreSQL 下部分 Hibernate 原生 SQL 参数绑定问题已通过**自定义 Repository 实现（动态条件 + EntityManager）**等方式解决；可按需关闭 show-sql 减少日志刷屏。

### 2. 待可选加强（非必须）

- **认证增强**：若课程大纲要求标准 JWT，可在现有学号密码登录基础上增加 **JWT 签发与拦截器**。
- **压测报告**：用 JMeter 整理 1000+ 并发下的 TPS、错误率、是否超卖（以数据库 left_stock、选课记录为准）形成截图与数据表写入报告。
- **运维向**：Redis 宕机降级策略、限流与防刷等（可作为扩展点描述）。

### 3. 原「Redis 未启动」类问题

当前阶段 **Redis 已纳入技术栈并完成联调**；若报告需保留过程说明，可改为：「中期曾遇 Redis 未启动导致连接异常，已通过本地启动 Redis 与配置 spring.redis.* 解决。」

## 五、小结

本项目已完成**从登录、抢课、退课到教务管理与导出**的完整链路，并通过 **Redis + 数据库一致性更新 + 业务防重** 体现 Web 高并发课程设计的核心目标。报告撰写时可将第四节「进展」作为**结题总结**，第三节作为**设计与实现依据**，并与仓库中的包结构、表名、接口路径保持一致，便于答辩演示与验收。
